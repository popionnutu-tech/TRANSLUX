# TRANSLUX — System Prompt (Final)

You are Claude Code 4.6. Build a production-ready internal system with:
1) Telegram bot ONLY for data entry by platform controllers (and invite activation).
2) Reporting/analytics ONLY in a WEB dashboard (no reporting inside Telegram).

Follow this spec strictly.

---

## LANGUAGE/UI

- Telegram bot UI in Romanian (RO). Keep texts short and operational.
- Web dashboard labels in RO.
- All date/time logic uses timezone **Europe/Chisinau** (UTC+2 / UTC+3 DST).

---

## CORE IDEA

- Two controller roles: **Controller Chișinău** (start point) and **Controller Bălți** (intermediate point).
- Controllers submit reports per trip per day through Telegram.
- Admin sees ALL reports ONLY in a web UI with filters by date range, route, direction, point, driver, status, and view mode (Passengers vs Compliance).
- Access for controllers ONLY via one-time invite link generated by Admin. The link auto-assigns point (CHIȘINĂU or BĂLȚI). Controllers cannot change their point.
- Drivers have NO access to bot; they exist only in the nomenclator.

---

## NOMENCLATORS

1. **Routes**: single nomenclator, same names everywhere.
2. **Drivers**: single list.
3. **Trips (Lista de curse)**: each trip is associated with:
   - `route_id`
   - `direction`: one of the enum values below
   - `time` (HH:MM)

Trips shown to controllers depend on their assigned point and direction.

**IMPORTANT**: after selecting a route, available trip times must be shown **sorted ascending**.

---

## DIRECTIONS (enum values)

```
CHISINAU_BALTI    — from Chișinău toward Bălți / North
BALTI_CHISINAU    — from Bălți toward Chișinău / South
```

These are the only two directions in the system. Every trip has exactly one direction.

---

## POINT / DIRECTION RULES

| Controller point | Reports for direction | What they report |
|---|---|---|
| CHIȘINĂU | `CHISINAU_BALTI` | Passengers departing from Chișinău going north |
| BĂLȚI | `BALTI_CHISINAU` | Passengers departing from Bălți going south |

- Reports are **independent by point** — each point reports only for its own direction.
- A single trip (e.g., 07:00 Chișinău→Bălți) generates **one report** from Controller Chișinău only.
- A single trip (e.g., 08:00 Bălți→Chișinău) generates **one report** from Controller Bălți only.

---

## REPORTING RULES

- Report is for a specific **date + point + trip**.
- Once a report is submitted (validated), it **cannot be repeated** for the same date + point + trip (`unique` constraint).
- A trip can be marked as **ABSENT** (did not occur):
  - If ABSENT: no passengers, no compliance booleans, no photos required.
  - If OK: require `passengers_count`, compliance fields, and photos.
- **Report cancellation**: a controller can cancel their own report **within 10 minutes** of submission. Cancelled reports are soft-deleted (`cancelled_at` timestamp) and free the unique slot so a corrected report can be submitted. Admin sees cancelled reports greyed out in dashboard.

## COMPLIANCE FIELDS

- `uniform_ok`: DA / NU

## PHOTOS

For status OK, photos are **mandatory**:
- Minimum **1 photo** of the driver.
- Photos should be captured on the spot, not from gallery.

**Enforcement (best-effort)**:
> Telegram Bot API cannot technically distinguish camera captures from gallery uploads. The following measures are best-effort deterrents, not guarantees:

1. Prompt the user to use the camera with clear instructions in Romanian.
2. Store upload timestamp for each photo.
3. The **first photo must arrive within 2 minutes** of entering the photo step; otherwise cancel the flow and require restart.
4. Log the `file_unique_id` to detect re-used photos across reports (flag duplicates in dashboard).

Store photos: `telegram_file_id` + storage key in **Supabase Storage**, associated with the report.

## SECURITY / AUTH

### Telegram Bot
- Only authorized Telegram IDs can interact.
- A Telegram ID becomes authorized **only** by consuming a valid invite token.
- **Rate limiting**: max 30 messages/minute per user. Exceeding → temporary 1-minute cooldown with message "Prea multe mesaje. Așteaptă un minut."

### Invites
- One-time token; after use → invalid.
- Store: `created_by` (admin), `created_at`, `used_at`, `expires_at`.

### Web
- Secure admin auth: email + password with **bcrypt** hashed passwords.
- Protect all API endpoints with auth middleware.
- Session-based or JWT auth (choose one, be consistent).

### Audit Logging
- Every report stores: `created_by`, `created_at`, `point`, `direction`.
- Invite usage logged: who used, when, from which telegram_id.
- Report cancellations logged: `cancelled_at`, `cancelled_by`.

## DATA MODEL (SQL migrations)

### Enums

```sql
CREATE TYPE user_role AS ENUM ('ADMIN', 'CONTROLLER');
CREATE TYPE point_enum AS ENUM ('CHISINAU', 'BALTI');
CREATE TYPE direction_enum AS ENUM ('CHISINAU_BALTI', 'BALTI_CHISINAU');
CREATE TYPE report_status AS ENUM ('OK', 'ABSENT');
```

### Tables

```sql
-- Users (controllers authorized via Telegram)
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  telegram_id BIGINT UNIQUE,
  username TEXT,
  role user_role NOT NULL DEFAULT 'CONTROLLER',
  point point_enum,
  active BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Admin accounts (web login)
CREATE TABLE admin_accounts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Invite tokens
CREATE TABLE invite_tokens (
  token TEXT PRIMARY KEY,
  role user_role NOT NULL DEFAULT 'CONTROLLER',
  point point_enum NOT NULL,
  created_by UUID NOT NULL REFERENCES admin_accounts(id),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at TIMESTAMPTZ NOT NULL,
  used_at TIMESTAMPTZ,
  used_by_user UUID REFERENCES users(id)
);

-- Routes
CREATE TABLE routes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT UNIQUE NOT NULL,
  active BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Drivers
CREATE TABLE drivers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  full_name TEXT NOT NULL,
  active BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Trips
CREATE TABLE trips (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  route_id UUID NOT NULL REFERENCES routes(id),
  direction direction_enum NOT NULL,
  departure_time TIME NOT NULL,
  active BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (route_id, direction, departure_time)
);

-- Reports
CREATE TABLE reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  report_date DATE NOT NULL,
  point point_enum NOT NULL,
  trip_id UUID NOT NULL REFERENCES trips(id),
  driver_id UUID REFERENCES drivers(id),
  status report_status NOT NULL,
  passengers_count INTEGER,
  exterior_ok BOOLEAN,
  uniform_ok BOOLEAN,
  created_by_user UUID NOT NULL REFERENCES users(id),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  cancelled_at TIMESTAMPTZ,
  cancelled_by UUID REFERENCES users(id),
  UNIQUE (report_date, point, trip_id)
);

-- Report photos
CREATE TABLE report_photos (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  report_id UUID NOT NULL REFERENCES reports(id) ON DELETE CASCADE,
  storage_key TEXT NOT NULL,
  telegram_file_id TEXT NOT NULL,
  file_unique_id TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### Indexes

```sql
CREATE INDEX idx_reports_date ON reports(report_date);
CREATE INDEX idx_reports_point ON reports(point);
CREATE INDEX idx_reports_driver ON reports(driver_id);
CREATE INDEX idx_reports_trip ON reports(trip_id);
CREATE INDEX idx_reports_status ON reports(status);
CREATE INDEX idx_reports_created_by ON reports(created_by_user);
CREATE INDEX idx_report_photos_report ON report_photos(report_id);
CREATE INDEX idx_trips_route_dir ON trips(route_id, direction);
CREATE INDEX idx_invite_tokens_expires ON invite_tokens(expires_at);
```

### Partial unique index (exclude cancelled reports)

```sql
CREATE UNIQUE INDEX idx_reports_unique_active
  ON reports(report_date, point, trip_id)
  WHERE cancelled_at IS NULL;
```

> Drop the plain `UNIQUE(report_date, point, trip_id)` constraint and use this partial unique index instead, so that cancelled reports free the slot.

## BOT FLOW (Telegram) — DATA ENTRY ONLY

### 1. `/start`

- If start param contains invite token:
  - Validate: exists, not expired (`expires_at > now()`), not used (`used_at IS NULL`).
  - Create or update user: `telegram_id`, `role=CONTROLLER`, `point` from token, `active=true`.
  - Mark invite as used: `used_at=now()`, `used_by_user=user.id`.
  - Reply: `"✓ Acces activat. Punctul tău: {CHIȘINĂU|BĂLȚI}."`
- Else if user is unauthorized:
  - Reply: `"Acces restricționat. Solicită un link de invitație de la Administrator."`
- Else if user is authorized controller:
  - Show main menu.

### 2. Main Menu

Buttons:
- `📋 Raportează cursă`
- `❓ Ajutor`

### 3. Raportează cursă (Report Flow)

State machine steps:

```
START → SELECT_ROUTE → SELECT_TIME → SELECT_DRIVER → ASK_STATUS
  ├─ ABSENT → SAVE_ABSENT → DONE
  └─ OK → ASK_PASSENGERS → ASK_EXTERIOR → ASK_UNIFORM → PHOTO_STEP → SAVE_OK → DONE
```

Details:

1. **Date**: today only (using `Europe/Chisinau` timezone). Display: `"Data: DD.MM.YYYY"`.
2. **SELECT_ROUTE**: list active routes as inline buttons. Paginate if > 8.
3. **SELECT_TIME**: list trip times for selected route + controller's direction, **sorted ascending**. Inline buttons, paginate if > 8.
4. **SELECT_DRIVER**: list active drivers. Support inline search by name (callback query filter). Paginate if > 8.
5. **ASK_STATUS**: `"Cursa a avut loc?"` → buttons `[DA]` `[NU - ABSENT]`
6. **If ABSENT**:
   - Check unique constraint (no active report for this date+point+trip).
   - Save report with `status=ABSENT`.
   - Reply: `"✓ Raport salvat: ABSENT."`
7. **If DA (OK)**:
   - **ASK_PASSENGERS**: `"Câți pasageri?"` — expect number input ≥ 0.
   - **ASK_EXTERIOR**: `"Exteriorul vehiculului este OK?"` → `[DA]` `[NU]`
   - **ASK_UNIFORM**: `"Șoferul poartă uniformă?"` → `[DA]` `[NU]`
   - **PHOTO_STEP**:
     - Send: `"📸 Fă o poză șoferului ACUM (folosește camera). Trimite minim 1 poză. Apasă Gata când ai terminat."`
     - Start 2-minute timer.
     - Accept photo messages; store each `file_id` + `file_unique_id` + timestamp.
     - Show `[Gata]` button after ≥ 1 photo received.
     - If 2 minutes pass with 0 photos → cancel: `"⏱ Timpul a expirat. Reîncepe raportul."`
   - **SAVE_OK**: check unique constraint → save report + photos → `"✓ Raport salvat: OK. Pasageri: {N}."`

### 4. Duplicate Detection

If active report exists for same date+point+trip:
- `"⚠ Deja există un raport pentru această cursă azi. Nu poate fi repetat."`

### 5. Cancel Own Report

Command or button: `"Anulează ultimul raport"` (available within 10 min of submission).
- Soft-delete: set `cancelled_at = now()`.
- Reply: `"Raportul a fost anulat. Poți retrimite."`

### 6. Rate Limiting

- Max 30 messages/minute per telegram_id.
- Exceeded → `"Prea multe mesaje. Așteaptă un minut."` + ignore for 60 seconds.
